# AI_LEARNING_SYSTEM

A complete, opinionated, operator-grade learning manual for working with modern AI systems. Built for people who want to stop guessing and start operating. No fluff, no hype, no emojis.

WEB VERIFICATION NOT AVAILABLE IN THIS RUN. Every platform-specific claim is tagged with a verification status. Treat untagged narrative as conceptual scaffolding, not live truth.

## How To Use This Manual

This manual is a system, not a textbook. Read it in order if you are new. Jump by role if you are already working. Use the cheat sheets daily once you have read the foundations.

| If You Want To... | Open This First |
|---|---|
| Start from zero | `00_START_HERE.md` |
| Understand what AI actually is | `01_WHAT_AI_IS.md` |
| Learn the vocabulary correctly | `02_AI_CORE_CONCEPTS.md`, `26_AI_GLOSSARY.md` |
| Write better prompts immediately | `03_PROMPT_ENGINEERING_FOUNDATION.md`, `04_PROMPT_VOCABULARY_LIBRARY.md` |
| Stop AI from lying to you | `06_HALLUCINATION_CONTROL.md`, `28_VERIFICATION_MATRIX.md` |

Use this manual the way a pilot uses a checklist. Read it, mark it up, and run drills.

## Verification Policy

Every platform-specific instruction in this manual is tagged. Read these as you would a NOTAM or a changelog.

| Tag | Meaning | Trust Level | When To Re-Verify |
|---|---|---|---|
| VERIFIED_LOCAL | Tested in this repo or on the author's machine | High | When the tool version changes |
| VERIFIED_DOCS | Confirmed from primary vendor documentation | High | Quarterly |
| USER_PROVIDED_UNVERIFIED | The user told us this; we did not test | Medium | Before relying on it |
| CUSTOM_WORKFLOW | A pattern we built, not a vendor feature | Medium | When the underlying tool changes |
| LIKELY_NATIVE_UNVERIFIED | Plausibly a vendor feature, not confirmed | Low | Before relying on it |
| UNKNOWN | We genuinely do not know | None | Always |
| DEPRECATED | Was true, no longer recommended | None | Do not use |

If a section has no tag, treat it as conceptual scaffolding, not a live command.

## Anti-Hallucination Policy

This manual is built on the following commitments. If you find a violation, file it as a defect.

1. No invented citations. If a source is named, it must exist and be traceable.
2. No fabricated commands. If a CLI flag, keyboard shortcut, or API parameter appears, it must carry a status tag.
3. No silent version drift. When vendor features change, the relevant cheat sheet is updated or tagged DEPRECATED.
4. No "trust me" claims. Statements that cannot be verified are flagged UNKNOWN or LIKELY_NATIVE_UNVERIFIED.
5. No hidden reasoning extraction. We never instruct readers to extract a model's private chain of thought. We use "reasoning summary" or "explain the decision path" only.
6. No emoji decoration. Signal density beats vibes.
7. No survivorship bias in examples. Failure cases are presented alongside success cases.
8. No mixing of fact and opinion. Opinions are labeled. Facts are tagged.

WEB VERIFICATION NOT AVAILABLE IN THIS RUN. Treat every external claim as a starting hypothesis until you verify it against the live vendor source.

## How To Contribute Updates

This is a living manual. When you discover that a tagged claim is now wrong, do the following.

1. Update the tag to DEPRECATED in place.
2. Add the new behavior in a sibling line tagged VERIFIED_DOCS or VERIFIED_LOCAL.
3. Add a row to `28_VERIFICATION_MATRIX.md` with the date and source.
4. Do not delete the old line for at least one quarter. Readers may still be running the old version.
